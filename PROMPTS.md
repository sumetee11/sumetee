# รวม Prompt ที่ดีที่สุดสำหรับระบบจัดการการแข่งขันหุ่นยนต์ (RobotComp)

เนื่องจากระบบนี้เป็นระบบเฉพาะทางที่ต้องรองรับทั้งกรรมการ (Judges) และผู้ดูแลระบบ (Admins) นี่คือชุดคำสั่ง (Prompts) ที่จะช่วยให้คุณพัฒนาและปรับปรุงระบบนี้ต่อได้อย่างมีประสิทธิภาพครับ

---

## 1. การจัดการข้อมูล (Data & Management)

### เพิ่มระบบ Import ทีมจาก Excel
> "เพิ่มฟีเจอร์ให้ Admin สามารถอัปโหลดไฟล์ Excel (.xlsx) เพื่อเพิ่มทีมในระบบพร้อมกันหลายๆ ทีม (Bulk Import) โดยให้ตรวจสอบความถูกต้องของฟิลด์ชื่อทีม, โรงเรียน, และระดับการแข่งขันก่อนบันทึกลง Firestore"

### ระบบรีเซ็ตคะแนนทั้งหมด (Reset System)
> "เพิ่มปุ่มในหน้า Dashboard ของ Admin สำหรับรีเซ็ตคะแนน (Scores) ทั้งหมดในระบบ เพื่อเริ่มการแข่งขันรอบใหม่ โดยต้องมี Modal ยืนยันการทำงานและให้เก็บ Log ไว้ว่าใครเป็นคนรีเซ็ต"

### การส่งออกผลการแข่งขัน (Export Results)
 > "เพิ่มฟีเจอร์ในหน้า Rankings ให้ Admin สามารถดาวน์โหลดสรุปผลการแข่งขันของทุกระดับเป็นไฟล์ PDF หรือ Excel ที่จัดรูปแบบสวยงาม พร้อมหัวกระดาษของรายการแข่งขัน"

---

## 2. ฟีเจอร์สำหรับการแข่งขัน (Competition Features)

### ระบบประกาศผลแบบ Real-time (Live Ticker)
> "สร้างคอมโพเนนต์ 'Live Score Ticker' ที่แถบด้านบนของหน้าแรก เพื่อเลื่อนแสดงผลคะแนนล่าสุดที่กรรมการเพิ่งบันทึกเข้ามา (Recent Scores) โดยดึงข้อมูลแบบ Real-time จาก Firestore"

### ระบบจับเวลาถอยหลัง (Competition Timer)
> "เพิ่มเครื่องมือจับเวลาถอยหลัง (Countdown Timer) ในหน้า Scoring สำหรับกรรมการ โดยให้ Admin สามารถตั้งเวลามาตรฐานของแต่ละประเภทการแข่งขันได้ และเมื่อหมดเวลาให้มีเสียงแจ้งเตือน"

### การพิมพ์เกียรติบัตร (Certificate Generation)
> "เพิ่มปุ่ม 'พิมพ์เกียรติบัตร' ในหน้า Rankings สำหรับทีมที่ได้อันดับ 1-3 โดยให้ระบบสร้างไฟล์ PDF ที่ระบุชื่อทีม, ชื่อโรงเรียน และรางวัลที่ได้รับอัตโนมัติ"

---

## 3. ความปลอดภัยและการตั้งค่า (Security & IT)

### จัดการ Firebase Rules (กรณีเปลี่ยนสิทธิ์)
> "อัปเดต firestore.rules ให้เข้มงวดขึ้น: กำหนดให้กรรมการ (Judge) สามารถแก้ไขคะแนน (Score) ได้เฉพาะรายการที่ตนเองเป็นคนสร้างเท่านั้น และห้ามแก้ไขข้อมูลทีมหรือระดับการแข่งขัน"

### แก้ไขปัญหา Connection Offline
> "ตรวจสอบระบบ Firebase Connection ใน src/firebase.ts และเพิ่ม Retry Logic หากการเชื่อมต่อล้มเหลว พร้อมแสดง Toaster notification แจ้งเตือนผู้ใช้เมื่อเน็ตหลุด"

---

## 4. ปรับเปลี่ยนดีไซน์ (UI/UX)

### ปรับแต่งธีมสีตามระดับ (Theme Customization)
> "ปรับปรุง UI ของหน้า Rankings ให้ใช้สีตามธีมของระดับการแข่งขัน (เช่น ประถมสีฟ้า, มัธยมต้นสีม่วง, มัธยมปลายสีส้ม) ให้ชัดเจนขึ้น และเพิ่ม Animation ตอนสลับอันดับให้ดู Smooth"

### โหมดมืด (Dark Mode)
> "เพิ่มระบบสลับ Dark Mode / Light Mode โดยใช้ Tailwind 'dark' class และให้ระบบจำสถานะไว้ใน LocalStorage"

---

## 5. การติดตั้ง (Prompt แรกเมื่อกู้คืนระบบ)
> "กรุณาตรวจสอบการตั้งค่า Firebase ของระบบนี้ และตรวจสอบว่าไฟล์ config (.json) ตรงกับโครงการใน Firebase Console หรือไม่ หากโครงการยังไม่มีคอลเลกชันเริ่มต้น ให้สร้างข้อมูล Mockup สำหรับ Competition Types และประเภทหุ่นยนต์เบื้องต้นมาให้ด้วย"
